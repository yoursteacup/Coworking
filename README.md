# Документация по совместной работе над проектом

## Репозиторий
 - Репозиторий проекта расположен по адресу https://github.com/yoursteacup/Coworking
 - Чтобы скопировать код на локальную машину:
`git clone git@github.com:yoursteacup/Coworking.git
cd Coworking`
 - После клонирования можно просматривать весь код в IDE или через командную строку.

## Методология работы
1. Пушить напрямую в main запрещено. Все изменения должны попадать в основную ветку через Pull Request из побочных веток;
2. Названия побочных веток должны содержательно и кратко описывать их конечную цель, например: fix/issue_93, ticket_lifespan_impl;
3. В качестве среды разработки рекомендуется PyCharm, но вы свободны использовать любую другую среду, которая вам ближе;
4. Обязательно настройте git среду прежде чем делать коммиты. Пример: https://github.com/yoursteacup/Coworking/blob/main/.gitconfig - примерное содержание ~/.gitconfig полученное в результате `git config --global user.name "Твоё Имя"`, `git config --global user.email "your_email@example.com"`;
5. Перед созданием pull request убедитесь, что качество кода соответствует стандарту (выполните ./code_quality.sh);
6. Перед созданием pull request убедитесь, что новые методы и логика имеют покрытие тестами;

## GIT шпаргалка 
 - Выбор ветки `git checkout <ИМЯ_ВЕТКИ`
 - Обновление ветки `git pull`
 - Создание новой ветки из текущей: `git checkout -b <ИМЯ_ВЕТКИ>`
 - Фиксация WIP: `git push -u origin <ИМЯ_ВЕТКИ>`
 - Добавление изменений в индекс изменений: `git add .`
 - Создание коммита изменений: `git commit -m "ОПИСАНИЕ КОММИТА"`
 - Отправка коммита: `git push`
 - Граф коммитов: `git history` / `git log --graph`
